# 🌿 Air Quality Data Analysis — India (2015–2020)

> A Python-based data analysis project exploring air pollution trends across Indian states and cities, with a strong emphasis on **data quality auditing** — catching and correcting sensor faults, fabricated imputation, and physically impossible readings before drawing any conclusions.

## 📌 Overview

This project analyzes daily air quality data across **26 Indian cities and 21 states (2015–2020)**, sourced from CPCB monitoring stations. Rather than trusting the dataset's columns at face value, the analysis runs a full **data quality audit first** — checking for missing coverage, physically impossible readings, and faulty sensors — and only then produces rankings and visualizations built on the metric proven to be reliable.

This audit step surfaced a real finding: the dataset's composite `AQI` column is distorted for several cities by faulty sensor readings (most notably Ahmedabad's CO), which would have produced a misleading "most polluted state" ranking if used directly. **PM2.5** was identified as the trustworthy metric instead and used throughout.

---

## 🎯 What This Project Does

1. **Data completeness audit** — flags every city that has zero real readings for a given pollutant (e.g., 14 of 26 cities have no Xylene data at all), so nothing gets silently fabricated by imputation
2. **Honest cleaning** — imputes missing values using only within-city medians; never fills a city's entirely-missing column from other cities' data
3. **Physical consistency checks** — flags impossible values (negative readings, PM2.5 > PM10, which violates the fact that PM2.5 is a physical subset of PM10) and duplicate records
4. **Anomaly detection with verdicts** — for every pollutant/city combination that looks statistically extreme, checks both mean and median to distinguish a **sustained sensor fault** (e.g., Ahmedabad's CO reads ~20x the national median, consistently) from a **spike-day artifact** (e.g., Shillong's benzene mean looks extreme, but its median is normal — a handful of bad days, not a real pattern)
5. **Top polluted states/cities**, ranked by PM2.5 (the metric proven reliable by the audit above), not the flawed composite AQI
6. **Seasonal pattern analysis** — quantifies the winter pollution spike
7. **10 visualizations** covering rankings, time-series trends, correlation, and seasonality

## 📊 What Gets Printed & Visualized

**Printed to console:**
- Dataset overview (record count, city/state count, date range)
- Data completeness audit (per city × pollutant)
- Cleaning summary (rows removed, cells imputed, cells genuinely left missing)
- Physical consistency check results (negatives, duplicates, PM2.5>PM10 rate)
- Anomaly report with sustained-vs-spike-artifact verdicts
- Top 5 / bottom 5 polluted states and cities (by PM2.5)
- Seasonal winter-vs-summer comparison

**Visualized (10 charts, shown inline via `plt.show()`):**
1. Top 5 most polluted states (PM2.5)
2. Top 5 most polluted cities (PM2.5)
3. Full 26-city PM2.5 ranking
4. National monthly PM2.5 trend, 2015–2020
5. Seasonal pattern (avg PM2.5 by month)
6. Metro city comparison (Delhi/Mumbai/Chennai/Bengaluru time series)
7. Pollutant correlation heatmap
8. City × month PM2.5 heatmap
9. State-wise pollution severity bubble chart
10. AQI category distribution

## 📁 Dataset Columns

| Column        | Description                                          |
| ------------- | ----------------------------------------------------- |
| `City`        | City name                                              |
| `Date`        | Date of reading                                        |
| `PM2.5`, `PM10` | Particulate matter (µg/m³)                           |
| `NO`, `NO2`, `NOx`, `NH3` | Nitrogen compounds (µg/m³)                 |
| `CO`          | Carbon monoxide (mg/m³)                                |
| `SO2`         | Sulfur dioxide (µg/m³)                                 |
| `O3`          | Ozone (µg/m³)                                          |
| `Benzene`, `Toluene`, `Xylene` | Volatile organic compounds (µg/m³)   |
| `AQI`         | Composite Air Quality Index (source-calculated)        |
| `AQI_Bucket`  | AQI category label (Good, Satisfactory, Poor, etc.)    |

`State` is derived by mapping each of the 26 cities to its state.

## 💡 Key Findings

- **Top 5 most polluted states by PM2.5**: Bihar (124.7), Delhi (117.5), Haryana (115.5), Uttar Pradesh (109.8), Gujarat (67.9) — consistent with the well-documented Indo-Gangetic Plain pollution belt
- **Winter PM2.5 is ~110% higher than summer**, driven by crop burning, temperature inversion, and reduced wind dispersal
- **The composite AQI column is unreliable for cross-city ranking** — it initially (and incorrectly) placed Gujarat #1, driven entirely by an anomalous CO reading in Ahmedabad (~20x the national median), not by its actual PM2.5/PM10 levels, which are close to the national average
- **17.63%** of all cells were missing before cleaning; **15.85%** of rows had no AQI value and were dropped as unusable for ranking
- **14 of 26 cities have zero real Xylene readings**, and several cities are missing entire pollutant columns (e.g., Lucknow has no PM10 data, Ahmedabad has no NH3 data) — these gaps are documented, not silently filled
- Several cities show **sustained anomalies** beyond Ahmedabad — Kochi (NO), Chennai (NH3), and unusually high benzene/toluene readings across Delhi, Kolkata, Amritsar, and others, suggesting VOC sensors may be miscalibrated more broadly in this dataset than a single-city issue

## ⚠️ Known Data Limitations

This project treats data quality as a first-class part of the analysis, not an afterthought. Readers should be aware:

- PM10 comparisons involving Lucknow, Chennai, Ahmedabad, Gurugram, and Patna should be treated with caution — real PM10 coverage for these cities ranges from 0–44%
- The composite `AQI` column should not be used for city/state ranking without first checking the anomaly report — see script output
- Benzene, Toluene, and Xylene readings are unreliable for roughly half the cities in the dataset due to missing or sparse coverage
